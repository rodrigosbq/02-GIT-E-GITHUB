Trabalhando com o Primeiro Repositorio GitHub


1- Procurar o icone com 3 bolinhas com o nome Controle de Código-GFonte ( Source Control), ou acessar pelo atalho crtl+shitf+G.
2- Clicar em initialize repository
3- Os arquivos deverão estar com um símbolo "U" ao lado do seu nome
o que quer dizer que eles ainda não estão com seu histórico de modificações monitorados
4- No controle de código fonte, ao lado de cada arquivo que desejar guardar o historico do arquivo clicar no botão "+".
5- O arquivo passará da lista Changes para a lista Staged Changes. Ou seja, estão separados para realizar o controle de versão.

6- Realizando um Commit , Deve ser escrita uma mensagem no Commit e os arquivos devem estar na lista Staged Changes.
7- Publicar arquivos: no Source Control, clicar em Publish branch, permitir o uso, confirmar a senha, escolher se o repositório será público ou privado.

8- O comando de enviar arquivos do seu PC para o GitHub é chamado de GitPush.
Algumas vezes após o primeiro envio do commit, é necessário fechar o VSCode, e realizar um novo commit a ativar novamente a autorização que pode ser por chave ou pelo proprio site.

9- O comando de receber arquivos do GitHub para o seu PC é chamado de GitPull.
Para baixar as ultimas atualizações do código do GitHub pode ser realizado abrindo o Terminal e digitando " git pull "
ou atraves do Source Control clicando nos 3 pontinhos (...) e clicando em Pull.

10- Comandos via Terminal:
git status: Verifica status dos arquivos do projetos e descreve para realizarmos um commit ou restaurar posição anterior.
git add nome do arquivo: Envia o arquivo para ser commitado.
git add . : Envia todos os arquivos modificados para o status Staged.
git commit -m"mensagem do commit": gera o commit para o github.
git push: envia os arquivos para o github.


SS- Extensões : Git Graph, na barra de baixo aparece o nome Gitgraph com os detalhamentos do commit.
Gitlens, mostra dentro do código , linha a linha clicada quem e quando foi realizada as alterações e habilita outras funções dentro do controle de código.
